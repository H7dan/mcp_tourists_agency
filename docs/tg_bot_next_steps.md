## Что такое Telegram Bot API (конкретно)

**API** здесь — это способ, которым твой код **общается с серверами Telegram по HTTP**.

- Telegram даёт боту **токен**. По этому токену Telegram понимает, что запросы идут от твоего бота.
- Твой скрипт **не подключается к пользователям напрямую**. Он только:
  - **шлёт HTTP-запросы** на адреса вида `https://api.telegram.org/bot<TOKEN>/метод` (например «получить обновления», «отправить сообщение»);
  - получает **JSON-ответ** (новые сообщения, результат отправки и т.д.).
- То есть **API** — это набор правил и URL: «какой запрос куда отправить и какой ответ придёт». Документация: [core.telegram.org/bots/api](https://core.telegram.org/bots/api).

Библиотека (python-telegram-bot или aiogram) **уже умеет** формировать эти HTTP-запросы и разбирать ответы. Ты вызываешь методы вроде `await bot.send_message(chat_id, "Hello")`, а библиотека сама делает запрос к `sendMessage` в Telegram Bot API.

## python-telegram-bot vs aiogram — что выбрать

| | **python-telegram-bot** (v20+) | **aiogram** (v3) |
|---|-------------------------------|------------------|
| **Стиль** | Классическая библиотека, много лет, стабильный API. | Более «современный» стек, сильнее завязан на типы и слои (Router, Middleware). |
| **Документация** | Очень подробная, много примеров на английском. | Хорошая, много примеров и туториалов на русском. |
| **Асинхронность** | Да (asyncio). | Да (asyncio), изначально проектировалась под async. |
| **Расширения** | Отдельные пакеты (например `python-telegram-bot[job-queue]`). | Много встроено (FSM, фильтры, middleware), экосистема вокруг aiogram. |
| **Кривая входа** | Чуть проще для минимального бота. | Чуть больше концепций (диспетчер, роутеры), зато удобно масштабировать. |
| **Сообщество / RU** | Больше EN. | Сильное русскоязычное сообщество. |

**Для нашего проекта лучше подойдёт aiogram:**

- Дальше будет интеграция с оркестратором (HTTP-запросы, возможно очереди, состояния диалога). У aiogram удобно встроены FSM (конечные автоматы для сценариев) и middleware — проще добавлять логирование, проверки, вызов оркестратора.
- Много материалов на русском.
- Один стиль асинхронного кода по всему боту.

Если важнее минимализм и «как в официальных примерах Telegram» — можно взять **python-telegram-bot**; для турагентства с диалогами и оркестратором логичнее **aiogram**.

## Что нужно для запуска базового бота (Hello World)

### 1. Получить токен бота

- Открой [@BotFather](https://t.me/BotFather) в Telegram.
- Команда: `/newbot`.
- Введи имя бота (например: `Tourists Agency Bot`) и username (должен заканчиваться на `bot`, например: `tourists_agency_bot`).
- BotFather пришлёт **токен** вида `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`.  
  **Это единственные данные, которые нужны для Hello World.**

Сохрани токен в переменную окружения или в файл `.env` (не коммить в репозиторий). В проекте уже есть `docker/.env.example` с полем `BOT_TOKEN=` — можно скопировать в `docker/.env` или в `services/tg_bot/.env` и подставить токен.

### 2. Выбор библиотеки

Выше в документе есть сравнение **python-telegram-bot** и **aiogram**. Для этого проекта рекомендуется **aiogram** (v3). Установка: `pip install aiogram`.

### 3. Минимальный код (идея)

- Точка входа: `services/tg_bot/main.py`.
- Зависимости: `services/tg_bot/requirements.txt` (например: `python-telegram-bot` или `aiogram`).
- В коде: создать приложение бота по токену, зарегистрировать обработчик на команду `/start` (и при желании на любое текстовое сообщение), в ответ отправить `Hello, World!` (или краткое приветствие). Запуск через `python main.py` или `asyncio.run(main())`.

### 4. Что понадобится позже (не для Hello World)

- **Оркестратор** — когда будешь подключать бота к LLM/MCP: URL оркестратора (например из `ORCHESTRATOR_URL` в `.env`), с бота — только HTTP-запросы к нему и отправка ответа пользователю.
- **База данных** — если понадобится хранить пользователей, сессии, историю; тогда доступ к БД (хост, порт, креды) из `docker/.env` или конфига сервиса.
- **Деплой** — для работы 24/7 бот должен крутиться на сервере или в контейнере; для разработки достаточно запуска на своей машине.

### 5. Чек-лист перед первым запуском

- [ ] Получен токен от @BotFather.
- [ ] Токен записан в `.env` (или в переменную окружения) как `BOT_TOKEN`.
- [ ] В `services/tg_bot/` добавлены `requirements.txt` и `main.py` с обработчиком `/start` и ответом "Hello, World!".
- [ ] Установлены зависимости (`pip install -r requirements.txt`), запуск `python main.py` из `services/tg_bot/`.
- [ ] В Telegram найден свой бот по username и отправлена команда `/start` — в ответ приходит приветствие.

После этого можно постепенно добавлять команды, интеграцию с оркестратором и остальной функционал.
